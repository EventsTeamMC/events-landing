Política de Privacidad

Última actualización: 22 de julio de 2026

Events Blacklist (el "Bot") es un bot de Discord que sincroniza baneos entre servidores y mantiene una base de datos compartida de baneos activos. Esta política explica exactamente qué datos maneja el Bot, cómo protege el origen de cada baneo y en qué caso excepcional ese origen puede consultarse.

1. Qué datos recopilamos

El Bot almacena únicamente lo necesario para sincronizar baneos:

  • ID de Discord del usuario baneado (nunca su nombre, avatar ni mensajes).
  • Por cada baneo: un identificador propio (ID de baneo), el motivo (si el baneo lo traía; si no, se guarda como "Sin razón"), la fecha y una marca de si es "sospechoso" (procedente de un servidor muy pequeño).
  • El origen del baneo —el servidor y el miembro del staff que lo ejecutó— guardado de dos formas: (a) un hash irreversible del servidor, para deduplicar y para estadísticas anónimas; y (b) una copia cifrada del origen, recuperable solo en el caso descrito en el punto 4.
  • La configuración que el staff de cada servidor define (canal de alertas, roles autorizados, umbral de aviso, auto-ban).
  • Un registro interno de auditoría de las acciones del sistema.

2. Qué NO hacemos

  • No leemos ni almacenamos el contenido de tus mensajes en ningún servidor (el Bot no usa el Message Content Intent).
  • No accedemos a canales privados ni a mensajes directos.
  • No vendemos ni compartimos datos con terceros ajenos al Bot.
  • No usamos los datos para publicidad ni para perfiles fuera del propio sistema de sincronización de baneos.

3. El origen de un baneo es anónimo para las comunidades

Ningún comando —ni público, ni de staff, ni de estadísticas— revela qué servidor originó un baneo concreto. El hash del servidor de origen es irreversible y solo sirve internamente para evitar duplicados y para contar "cuántas comunidades distintas han baneado a este usuario". Para tu comunidad y para el resto, quién baneó a quién es siempre anónimo. Así, reportar no expone a tu servidor a represalias.

4. Acceso excepcional del operador para auditar abusos

Además del hash irreversible, el origen de cada baneo (servidor y staff) se guarda cifrado. Este dato puede descifrarlo únicamente el superusuario que opera la infraestructura del Bot, mediante una herramienta interna de administración, y solo para investigar posibles abusos del sistema —por ejemplo, un servidor que registra baneos falsos de forma masiva para perjudicar a alguien—. Cada uno de estos accesos queda registrado en la auditoría del sistema. Este mecanismo no está disponible para ningún servidor ni para ningún miembro de staff: existe exclusivamente para mantener la red limpia y justa.

5. Para qué usamos los datos

  • Sincronizar los baneos entre los servidores activos de la red.
  • Avisar al staff de un servidor cuando entra un usuario con baneos activos.
  • Aplicar auto-ban en los servidores que lo hayan activado.
  • Mostrar estadísticas agregadas y anónimas (nunca desglosadas por servidor).
  • Auditar el sistema y prevenir abusos, según el punto 4.

6. Cuánto tiempo conservamos los datos

Un baneo permanece registrado mientras esté activo. Si un servidor desbanea a un usuario, su baneo correspondiente se elimina de la base de datos. Si el Bot es expulsado de un servidor, todos los baneos originados en él se eliminan. El superusuario también puede retirar manualmente un baneo concreto.

7. Tus derechos

Si el Bot tiene un registro sobre tu cuenta de Discord, puedes:

  • Pedir información sobre qué datos tenemos asociados a tu ID de Discord.
  • Solicitar la eliminación de tus datos escribiendo al contacto indicado más abajo, salvo que exista un motivo legítimo para conservarlos (por ejemplo, un baneo activo en un servidor).

8. Seguridad

El origen cifrado de los baneos se protege con una clave que solo controla quien opera la infraestructura del Bot, y el acceso a las herramientas de administración está restringido al superusuario. Aun así, ningún sistema es infalible al 100%; si detectas una vulnerabilidad, contáctanos.

9. Menores de edad

Events Blacklist sigue los mismos requisitos de edad mínima que Discord. No solicitamos ni verificamos activamente la edad de los usuarios más allá de lo que la propia plataforma de Discord ya exige.

10. Cambios en esta política

Si actualizamos esta política de forma relevante, lo anunciaremos a través de los canales oficiales del Bot. La fecha de "última actualización" en esta página siempre refleja la versión vigente.

11. Contacto

Para cualquier duda sobre privacidad o para ejercer tus derechos, escribe a support@eventsmc.xyz.
